# B1: Tiny Generative SGS Model — Plan

## Concept

Train a small language model that generates text using SGS rendering instead of transformer attention. Not aiming for GPT quality — aiming for "it works, it's novel, it cost $X."

## What It Would Do

Input a prompt → SGS model generates text token by token via autoregressive rendering.

Each generation step:
1. The current tokens form a Gaussian scene
2. Render the scene from a "next-token query viewpoint"
3. The rendered meaning vector is projected to vocabulary → next token
4. New token's Gaussian is added to the scene
5. Repeat

This is the architecture described in the whitepaper (Section 5.8) but never implemented for actual text generation.

## Architecture

```
SGSLanguageModel:
  Vocabulary: 30K Gaussians (BPE tokenizer, not GloVe)
  Splatting space: d_s = 64
  Feature space: d_f = 256
  Passes: 2
  Context: up to 512 tokens (Gaussians in the scene)

  forward(token_ids):
    For each position t:
      1. Activate Gaussians for tokens 1..t-1
      2. Positional modulation (position embeddings)
      3. Multi-pass rendering (2 passes)
      4. Query at position t → rendered meaning vector
      5. Project to vocab: logits = meaning @ vocab_features.T
      6. Loss = cross_entropy(logits, token_ids[t])
```

## Key Differences from STS-B Encoder

| Property | STS-B SGS | Generative SGS |
|---|---|---|
| Tokenizer | GloVe word-level | BPE (tiktoken or sentencepiece) |
| Training objective | Cosine similarity | Next-token prediction (causal LM) |
| Query | Centroid of all tokens | Learned position-dependent query |
| Initialization | GloVe pre-trained | Random (learned from scratch) |
| Context | Full sentence (bidirectional) | Causal (left-to-right only) |

## Training Data

**Option 1: TinyStories** (Eldan & Li, 2023)
- ~2.1M short children's stories generated by GPT-3.5/4
- Simple vocabulary, short sentences
- Known to produce coherent small models (~30M params can generate readable stories)
- This is how Microsoft trained their tiny language models

**Option 2: WikiText-103**
- ~100M tokens of Wikipedia
- Standard language modeling benchmark
- More complex vocabulary and structure

**Recommendation: TinyStories.** If a 30M-param SGS model can generate coherent children's stories, the demo is compelling regardless of quality vs GPT.

## Training Plan

| Parameter | Value |
|---|---|
| Vocab size | 30,000 (BPE) |
| d_s | 64 |
| d_f | 256 |
| n_passes | 2 |
| Context length | 256 tokens |

| Hardware | RTX 4090 |

| Estimated cost | ~$5-10 (electricity) |

## What Success Looks Like

**Minimum:** Model generates grammatically correct short sentences that are roughly on-topic. Even if quality is below a matched transformer, the fact that a "rendering-based language model" generates text at all is novel.

**Good:** Model generates coherent short stories (3-5 sentences) from TinyStories. Perplexity within 2x of a matched-parameter transformer.

**Great:** Model generates text of comparable quality to a matched transformer, demonstrating that rendering-based composition doesn't lose generation ability. The "$10 language model trained with 3D rendering math" hook becomes credible.

## Comparison Baseline

Train a vanilla Transformer LM with the same parameter count (~30M) on the same data. This is the only fair comparison — not GPT-4, not BERT, just a matched transformer.

## Risks

1. **Autoregressive rendering is slow.** Each token requires a full rendering pass over the growing scene. With 256 tokens and 2 passes, that's 512 rendering operations per sequence. May need caching.

2. **Causal masking in rendering.** The rendering equation naturally composes left-to-right (transmittance is ordered), which is already causal. But the multi-pass updates need to be masked to prevent looking ahead.

3. **BPE tokens don't have pre-trained Gaussian positions.** Unlike GloVe-initialized STS-B, the generative model learns Gaussian parameters from scratch. The inductive bias advantage may be weaker without good initialization.

4. **Quality will be low by modern standards.** 30M parameters on a single GPU cannot compete with 7B+ models. The framing must be "novel mechanism, small scale" not "competitive model."

## The Hook (If It Works)

> "We trained a language model that generates text by rendering sentences like 3D scenes. The same equation that turns Gaussian blobs into photorealistic images turns word distributions into coherent text. It cost $10 to train on a single RTX 4090."

This is shareable regardless of quality, as long as the generated text is readable.
